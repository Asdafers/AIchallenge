# Research Synthesis

This file summarizes external AI research inputs from Gemini, Claude, and NotebookLM-style exports. Treat it as strategy input, not final factual authority.

## Durable Decisions

- Keep the working product name as Methodic.
- Position as an autonomous research operations agent, not an AI survey tool.
- Default to Track 1.
- Use a B2B SaaS win-loss scenario as the demo wedge.
- Build the demo around clean data capture, not analytics.
- Make the output evidence-linked, structured, and quality-scored.
- Use a polished but focused vertical slice rather than a broad platform.

## Strongest Differentiators

- Decision in, governed data out.
- Methodology Agent pushes back on weak plans.
- Survey Agent performs adaptive clarification while preserving measurement intent.
- MCP triangulation enriches a participant answer with approved business context.
- A2A-style external agent request makes Methodic feel native to an agentic enterprise.
- Data Quality Agent links every structured claim to transcript evidence.
- Saturation or coverage curves show when enough useful data has been captured.

## Methodology To Incorporate

- Survey response cognition: questions can fail at comprehension, retrieval, judgment, or response.
- Satisficing: participants often provide shallow or low-effort answers unless the instrument detects and adapts.
- Cognitive interviewing: test whether participants understand questions the way the organizer intends.
- Theoretical saturation: show when new sessions stop adding new themes, but avoid over-claiming this from tiny samples.
- Qualitative trustworthiness: use audit trails, dependability, and confirmability as rationale for evidence-linked data.
- Inter-coder reliability: use Krippendorff's alpha only if implemented honestly.

## Claims To Avoid

- Statistically representative unless the sample design supports it.
- Causal insight unless there is an experiment.
- Bias elimination.
- Fully replacing human researchers.
- Synthetic respondents as a replacement for real research.
- Better than human moderators in all respects.

## Build Implications

- Build methodology pushback before broad flexibility.
- Hard-code a small number of high-quality critique patterns first: sampling mismatch, leading question, double-barreled question, cognitive overload, and missing economic buyer.
- Build one memorable MCP triangulation path rather than many shallow integrations.
- Build a developer overlay so judges can see ADK handoffs and MCP calls.
- Use simulated participant personas for the demo.
- Export to BigQuery or a BI-ready table; do not build a full analytics dashboard.
